## 1. 两条并行流式路径

DeerFlow 存在两条完全独立的流式路径，两者**无法合并**：

| 维度 | Gateway 路径 | DeerFlowClient 路径 |
|------|--------------|---------------------|
| 入口 | `POST /api/threads/{id}/runs/stream` | `DeerFlowClient.stream(message)` |
| 执行模型 | `async def` + `agent.astream()` | sync generator + `agent.stream()` |
| 事件传输 | `StreamBridge`（asyncio Queue）+ SSE | 直接 `yield` |
| 序列化 | JSON dict，匹配 LangGraph Platform wire 格式 | 原生 LangChain 对象 |
| 消费者 | 浏览器、IM 渠道、LangGraph SDK 客户端 | Jupyter notebook、测试脚本 |
| 断连恢复 | `Last-Event-ID` SSE 重连 | 无需要 |

## 2. LangGraph stream_mode 三层语义

LangGraph 的 `agent.stream(stream_mode=[...])` 是**多路复用**接口，同时订阅多个独立事件源：

| 模式 | 触发时机 | 内容 |
|------|----------|------|
| `values` | 每个节点执行完成后 | 完整 State 快照 |
| `messages` | 每个 LLM token | token 级别 delta |
| `custom` | 显式 `StreamWriter` 调用 | 自定义事件（如子代理状态）|

> ⚠️ **三种模式是独立事件源，不是详细程度的梯度**。要获取 token 流必须显式订阅 `messages` 模式。

```python
# 订阅全部三种模式
async for chunk in agent.astream(
    input,
    stream_mode=["values", "messages", "custom"]
):
    event_type, data = chunk
    if event_type == "messages":
        # AIMessageChunk - token 级别流
        pass
    elif event_type == "values":
        # 完整 State 快照
        pass
    elif event_type == "custom":
        # 自定义事件（子代理状态等）
        pass
```

## 3. Gateway 流式路径（HTTP SSE）

```
POST /api/threads/{id}/runs/stream
    │
    ▼
RunManager.create_run()
    │
    ▼ 后台 asyncio 任务
run_agent()
    │
    ├── agent.astream(stream_mode=["values", "messages", "custom"])
    │
    └── 每个 chunk → serialize(chunk) → StreamBridge（asyncio.Queue）
                                              │
                                              ▼
                                    sse_consumer → SSE 事件流 → 客户端
```

### SSE 事件格式

```
data: {"event": "messages/partial", "data": {"id": "...", "content": [...]}}

data: {"event": "values", "data": {"messages": [...], "artifacts": [...]}}

data: {"event": "custom/task_started", "data": {"task_id": "...", "description": "..."}}

data: {"event": "custom/task_completed", "data": {"task_id": "...", "result": "..."}}
```

### 断连恢复

客户端可以发送 `Last-Event-ID` 请求头重新连接，`StreamBridge` 会从上次位置继续推送事件。

## 4. DeerFlowClient 路径（内嵌 Python）

**文件**：`backend/packages/harness/deerflow/client.py`

```python
from deerflow.client import DeerFlowClient, StreamEvent

client = DeerFlowClient()

# 同步流式调用
for event in client.stream("帮我分析 Python 代码质量"):
    match event.type:
        case "message":
            # AIMessage 片段
            print(event.data.content, end="", flush=True)
        case "tool_call":
            # 工具调用
            print(f"\n[工具] {event.data.name}({event.data.args})")
        case "tool_result":
            # 工具结果
            print(f"\n[结果] {event.data.content}")
        case "values":
            # State 快照
            state = event.data
        case "task_started":
            print(f"\n[子代理] 开始: {event.data.description}")
        case "task_completed":
            print(f"\n[子代理] 完成")
```

### 三个 ID Set 的不变式

`DeerFlowClient.stream()` 内部维护三个 `set[str]`，管理三个独立不变式：

| Set | 作用 |
|-----|------|
| `seen_ids` | 确保每条消息只处理一次（去重）|
| `streamed_ids` | 确保 token 流只对每条消息流式处理一次 |
| `counted_usage_ids` | 确保 token 使用量只计数一次 |

> ⚠️ 这三个 Set 看起来相似但管理的是三个独立不变式，**不能合并**。

## 5. 自定义事件（StreamWriter）

在中间件或工具中发出自定义 SSE 事件：

```python
from langchain_core.runnables.config import get_callback_manager_for_config

# 在中间件中发出自定义事件
async def my_middleware_hook(state: ThreadState, config: RunnableConfig):
    # 通过 StreamWriter 发出自定义事件
    from langgraph.config import get_stream_writer
    
    writer = get_stream_writer()
    if writer:
        await writer({
            "type": "my_custom_event",
            "data": {"key": "value"}
        })
```

## 6. 前端接收 SSE 事件

DeerFlow 前端使用 `useStream` hook（`frontend/src/core/threads/`）接收 SSE 事件：

```typescript
// 核心 SSE 消费逻辑（简化）
const response = await fetch('/api/threads/{threadId}/runs/stream', {
  method: 'POST',
  body: JSON.stringify(runRequest)
});

const reader = response.body!.getReader();
const decoder = new TextDecoder();

for await (const chunk of readSSE(reader, decoder)) {
  const event = JSON.parse(chunk.data);
  
  switch (event.event) {
    case 'messages/partial':
      // 更新消息内容
      updateMessage(event.data);
      break;
    case 'values':
      // 更新 State（artifacts、todos 等）
      updateState(event.data);
      break;
    case 'custom/task_started':
      // 显示子代理任务开始
      showTaskStarted(event.data);
      break;
  }
}
```

## 7. 流式输出测试

```python
# 使用 DeerFlowClient 进行集成测试
from deerflow.client import DeerFlowClient

def test_streaming():
    client = DeerFlowClient()
    events = list(client.stream("计算 1+1"))
    
    # 验证至少有一条消息事件
    message_events = [e for e in events if e.type == "message"]
    assert len(message_events) > 0
    
    # 验证最终答案包含 "2"
    final_content = "".join(
        e.data.content for e in message_events 
        if hasattr(e.data, 'content')
    )
    assert "2" in final_content
```
