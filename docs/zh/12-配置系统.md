## 1. 两个配置文件

| 文件 | 内容 | 位置 |
|------|------|------|
| `config.yaml` | 模型、工具、沙箱、数据库、记忆等核心配置 | 项目根目录（推荐）|
| `extensions_config.json` | MCP 服务器列表、Skills 启用状态 | 项目根目录（推荐）|

### 配置文件查找优先级（两个文件均适用）

```
1. 显式传入的 config_path 参数
2. 环境变量（DEER_FLOW_CONFIG_PATH / DEER_FLOW_EXTENSIONS_CONFIG_PATH）
3. 当前目录（backend/）中的配置文件
4. 父目录（项目根）中的配置文件  ← 推荐位置
```

## 2. config.yaml 完整结构

```yaml
# 配置版本（用于检测是否过期）
config_version: 1

# =====================
# 模型配置
# =====================
models:
  - name: gpt-4o                        # 内部唯一标识符
    display_name: GPT-4o                # 前端显示名
    use: langchain_openai:ChatOpenAI    # LangChain 类路径
    model: gpt-4o                       # API 模型标识
    api_key: $OPENAI_API_KEY            # 环境变量引用
    max_tokens: 4096
    temperature: 0.7
    supports_vision: false              # 是否支持图像
    supports_thinking: false            # 是否支持 Extended Thinking
    when_thinking_enabled: {}           # thinking 开启时的覆盖参数

# =====================
# 工具配置
# =====================
tools:
  - name: tavily_search
    use: deerflow.community.tavily:TavilySearchTool
    enabled: true
    config:
      api_key: $TAVILY_API_KEY

# =====================
# 沙箱配置
# =====================
sandbox:
  use: local              # local | aio | provisioner

# =====================
# 数据库配置（需重启生效）
# =====================
database:
  connection_string: sqlite:///backend/.deer-flow/deerflow.db

# =====================
# 记忆系统配置
# =====================
memory:
  enabled: true
  max_injection_tokens: 2000

# =====================
# 上下文摘要配置
# =====================
summarization:
  enabled: false
  max_tokens: 8000         # 超过此 Token 数触发摘要

# =====================
# 标题自动生成配置
# =====================
title:
  enabled: true
  model:
    name: gpt-4o-mini
    use: langchain_openai:ChatOpenAI
    api_key: $OPENAI_API_KEY

# =====================
# 子代理配置
# =====================
subagents:
  enabled: true
  max_concurrent: 3
  timeout_minutes: 15

# =====================
# 安全护栏配置（可选）
# =====================
guardrails:
  enabled: false
  provider:
    use: deerflow.guardrails.allowlist:AllowlistProvider
    config:
      allowed_tools: ["bash", "read_file", "write_file"]

# =====================
# 日志级别（需重启生效）
# =====================
log_level: INFO

# =====================
# IM 渠道配置（需重启生效）
# =====================
channels:
  feishu:
    enabled: false
    app_id: $FEISHU_APP_ID
    app_secret: $FEISHU_APP_SECRET
```

## 3. 热重载 vs 需要重启的字段

### 热重载字段（修改后下次请求生效）

- `models[*].max_tokens`、`temperature` 等模型参数
- `summarization.*`
- `title.*`
- `memory.*`
- `subagents.*`
- `tools[*]`
- Agent 系统提示词相关配置

### 需要重启的字段

| 字段 | 原因 |
|------|------|
| `database.*` | SQLAlchemy 引擎在启动时初始化，持有连接池 |
| `checkpointer.*` | Checkpointer 在启动时绑定 |
| `run_events.*` | 在启动时选择内存/SQL 实现 |
| `stream_bridge.*` | Bridge 对象在启动时构建 |
| `sandbox.use` | Provider 单例在启动时缓存 |
| `log_level` | 根 Logger 级别在启动时设置 |
| `channels.*` | IM 渠道服务在启动时初始化 |

## 4. 环境变量解析

所有 `$` 开头的值自动解析为环境变量：

```yaml
api_key: $OPENAI_API_KEY      # → os.environ["OPENAI_API_KEY"]
```

若环境变量未设置，启动时会报错。

## 7. 开发步骤：添加新的配置字段

### 步骤 1：在 `config.example.yaml` 中添加字段

```yaml
# config.example.yaml
my_feature:
  enabled: false
  max_items: 10
  api_key: $MY_API_KEY
```

同时更新 `config_version`（递增 1）。

### 步骤 2：在 `deerflow/config/` 中定义 Pydantic 模型

```python
# backend/packages/harness/deerflow/config/models.py
from pydantic import BaseModel

class MyFeatureConfig(BaseModel):
    enabled: bool = False
    max_items: int = 10
    api_key: str = ""
```

### 步骤 3：在 `AppConfig` 中注册

```python
class AppConfig(BaseModel):
    # 现有字段...
    my_feature: MyFeatureConfig = MyFeatureConfig()
```

### 步骤 4：在代码中使用

```python
from deerflow.config import get_app_config

config = get_app_config()
if config.my_feature.enabled:
    # 功能逻辑
    pass
```
