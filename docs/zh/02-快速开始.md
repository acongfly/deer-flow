## 1. 环境要求

| 工具 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | 22+ | 前端运行时 |
| pnpm | 10+ | 前端包管理器 |
| Python | 3.12+ | 后端运行时 |
| uv | 最新 | Python 包管理器（替代 pip/poetry）|
| nginx | 最新 | 本地反代（`make dev` 需要）|
| Docker Desktop | 最新 | 可选，用于 Docker 开发模式 |

运行检查命令：
```bash
make check
```

## 2. 首次安装

### 2.1 克隆仓库

```bash
git clone https://github.com/bytedance/deer-flow.git
cd deer-flow
```

### 2.2 生成配置文件

```bash
make config
```

这将基于 `config.example.yaml` 在项目根目录生成 `config.yaml`。

> ⚠️ `make config` 是非幂等的：若 `config.yaml` 已存在，命令会中止。

### 2.3 配置模型 API Key

编辑 `config.yaml`，填写你要使用的模型 API Key：

```yaml
models:
  - name: gpt-4o
    display_name: GPT-4o
    use: langchain_openai:ChatOpenAI
    model: gpt-4o
    api_key: $OPENAI_API_KEY   # 使用环境变量
    max_tokens: 4096
```

或直接导出环境变量：
```bash
export OPENAI_API_KEY="sk-..."
```

推荐模型组合：
- Doubao-Seed-2.0-Code（字节豆包）
- DeepSeek v3.2
- Kimi 2.5
- GPT-4o / Claude 3.7 Sonnet

### 2.4 安装依赖

```bash
make install
```

这会依次安装后端依赖（uv）和前端依赖（pnpm），并设置 pre-commit hooks。

## 3. 运行方式

### 方式一：本地开发（推荐快速调试）

```bash
make dev
```

这会同时启动：
- LangGraph 运行时（内嵌于 Gateway，端口 8001）
- Gateway API（端口 8001）
- Frontend（端口 3000）
- Nginx（端口 2026，统一入口）

日志文件：
```
logs/langgraph.log
logs/gateway.log
logs/frontend.log
logs/nginx.log
```

停止所有服务：
```bash
make stop
```

访问地址：<http://localhost:2026>

### 方式二：Docker 开发（推荐协作 / PR Review）

```bash
# 第一次初始化（构建镜像、安装依赖）
make docker-init

# 启动服务
make docker-start

# 查看日志
make docker-logs

# 停止服务
make docker-stop
```

Docker 架构：
```
Host Machine
  ↓
Docker Compose (deer-flow-dev)
  ├→ nginx（端口 2026）← 反向代理
  ├→ web（端口 3000）← 前端，热重载
  ├→ gateway（端口 8001）← Gateway API + Agent 运行时，热重载
  └→ provisioner（可选，端口 8002）← 仅 provisioner/K8s sandbox 模式启动
```

访问地址：<http://localhost:2026>

#### Docker 常用命令

```bash
make docker-init          # 构建镜像 + 安装依赖（仅第一次）
make docker-start         # 启动服务
make docker-stop          # 停止服务
make docker-logs          # 查看所有服务日志
make docker-logs-frontend # 查看前端日志
make docker-logs-gateway  # 查看 Gateway 日志
```

## 4. 手动分服务启动（高级）

```bash
# Terminal 1：启动 Gateway（含内嵌 Agent 运行时）
cd backend && make dev

# Terminal 2：启动前端
cd frontend && pnpm dev

# Terminal 3：启动 nginx
make nginx
# 或直接：nginx -c $(pwd)/docker/nginx/nginx.local.conf -g 'daemon off;'
```

## 5. 推荐资源配置

| 场景 | 起步配置 | 推荐配置 |
|------|----------|----------|
| 单机 `make dev` | 4 vCPU / 8 GB RAM | 8 vCPU / 16 GB RAM |
| Docker 开发 | 4 vCPU / 8 GB RAM | 8 vCPU / 16 GB RAM |
| 共享 Linux 测试服务器 | 8 vCPU / 16 GB RAM | 16 vCPU / 32 GB RAM |

> ⚠️ 2 vCPU / 4 GB 的环境在正常 DeerFlow 负载下极易启动失败或无响应。

## 6. 配置升级

当 `config.example.yaml` 的 `config_version` 比你本地 `config.yaml` 高时，启动时会提示：

```
WARNING - Your config.yaml (version 0) is outdated — the latest version is 1.
Run `make config-upgrade` to merge new fields into your config.
```

运行以下命令自动合并缺失字段（保留你的现有值，并生成 `.bak` 备份）：

```bash
make config-upgrade
```

## 8. Linux Docker 权限问题

若在 Linux 上遇到 `permission denied` 连接 Docker daemon：

```bash
# 查看 docker 组是否存在
getent group docker

# 添加当前用户到 docker 组
sudo usermod -aG docker $USER

# 刷新组（最可靠方式：重新登录）
newgrp docker

# 验证
docker ps
```
