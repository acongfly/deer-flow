## 3. 关键模块：threads（对话线程）

`frontend/src/core/threads/` 是前端最核心的模块，负责管理与后端 Agent 的对话状态。

```typescript
// 核心功能
- 创建/删除线程
- 发送消息（触发 Agent 运行）
- 接收 SSE 流（useStream hook）
- 管理消息历史
- 处理工具调用展示
- 管理 Agent 状态（thinking/running/idle）
```

## 4. 关键模块：api（API 请求封装）

`frontend/src/core/api/` 封装了所有与 Gateway 的 HTTP 通信：

```typescript
// 示例 API 调用模式
import { apiClient } from '@/core/api';

// 发送消息并获取 SSE 流
const stream = await apiClient.threads.runs.stream(threadId, {
  assistant_id: 'agent',
  input: { messages: [{ role: 'user', content: message }] },
  config: { configurable: { model_name: selectedModel } }
});
```

## 5. 环境变量（env.js）

**文件**：`frontend/src/env.js`（使用 `@t3-oss/env-nextjs`）

```javascript
// 关键环境变量
BETTER_AUTH_SECRET         // 认证密钥（构建时必须）
NEXT_PUBLIC_API_URL        // API 基础 URL（可选，默认通过 nginx 代理）
SKIP_ENV_VALIDATION        // 跳过环境验证（仅开发时）
```

> ⚠️ **构建时必须设置 `BETTER_AUTH_SECRET`**，否则 build 会失败。

```bash
# 正确的构建方式
BETTER_AUTH_SECRET=local-dev-secret pnpm build

# 或跳过验证（不推荐）
SKIP_ENV_VALIDATION=1 pnpm build
```

## 8. 开发步骤：添加新页面

### 步骤 1：创建路由文件

```
frontend/src/app/my-feature/page.tsx
frontend/src/app/my-feature/layout.tsx  # 可选
```

### 步骤 2：创建页面组件

```tsx
// frontend/src/app/my-feature/page.tsx
import { MyFeatureContent } from '@/components/my-feature';

export default function MyFeaturePage() {
  return (
    <main>
      <MyFeatureContent />
    </main>
  );
}
```

### 步骤 3：添加 API 调用

```typescript
// frontend/src/core/api/my-feature.ts
export async function fetchMyFeatureData() {
  const response = await fetch('/api/my-feature');
  return response.json();
}
```

## 10. 国际化（i18n）

`frontend/src/core/i18n/` 管理多语言支持。目前支持：
- 中文（zh）
- 英文（en）
- 日语（ja）
- 法语（fr）
- 俄语（ru）

添加新语言：在 i18n 目录下添加对应的语言文件并注册。
