## 3. 运行时组件关系

| 组件 | 端口 | 职责 |
|------|------|------|
| Nginx | 2026 | 统一入口，路由到 Gateway / Frontend |
| Gateway (FastAPI) | 8001 | REST API + LangGraph-compatible 运行时 + SSE 流 |
| Frontend (Next.js) | 3000 | Web UI |
| Provisioner (可选) | 8002 | 仅在 provisioner/K8s sandbox 模式下启动 |

## 4. 核心设计决策

### 4.1 Harness / App 分层

- **Harness**（`packages/harness/deerflow/`）：设计为可独立发布的 Agent 框架包，包含 Agent 编排、工具、沙箱、模型、MCP、Skills、配置等所有运行 Agent 所需的东西。
- **App**（`app/`）：不发布，是应用层，包含 FastAPI Gateway API 和 IM 渠道集成（飞书、Slack、Telegram、钉钉）。
- **依赖规则**：App 导入 deerflow；deerflow **绝不**导入 app。这个边界由 `tests/test_harness_boundary.py` 在 CI 中强制执行。

### 4.2 LangGraph 嵌入 vs 独立服务

Agent 运行时**内嵌在 Gateway 中**，通过 `RunManager` + `run_agent()` + `StreamBridge` 驱动（位于 `packages/harness/deerflow/runtime/`）。Nginx 把 `/api/langgraph/*` 重写到 Gateway 的原生路由，保持与 LangGraph SDK 客户端的兼容性，无需单独运行 LangGraph Server。

`langgraph.json` 仅保留用于 LangGraph Studio 工具调试。

### 4.3 配置热重载

`get_app_config()` 在每次请求时检测 `config.yaml` 文件的 mtime 是否变化，如有变化则自动重新加载。大多数运行时参数（模型、记忆、摘要、标题等）无需重启即可生效。数据库连接池、Checkpointer、Sandbox 提供者等基础设施字段需要重启。

## 5. 数据流概览

```
用户消息（HTTP POST /api/threads/{id}/runs/stream）
    │
    ▼
Gateway auth / CSRF 中间件
    │
    ▼
RunManager.create_run()
    │
    ▼
run_agent() [后台 asyncio 任务]
    │
    ├─▶ make_lead_agent(config)
    │       │
    │       ▼
    │   中间件链（18 层，详见 Agent 系统文档）
    │       │
    │       ▼
    │   LLM 调用（tool_calls / text）
    │       │
    │       ▼
    │   工具执行（sandbox / MCP / built-in / subagent）
    │       │
    │       └─▶ 子代理（background thread，最多 3 并发）
    │
    └─▶ StreamBridge → SSE 事件 → 浏览器 / IM 渠道
```
