## 1. 开发工作流

### 1.1 分支管理

```bash
# 从 main 分支创建 feature 分支
git checkout main
git pull origin main
git checkout -b feature/your-feature-name

# 或 bug fix 分支
git checkout -b fix/bug-description
```

### 1.2 每次变更前的检查

```bash
# 1. 确认系统环境
make check

# 2. 确认依赖是最新的
make install

# 3. 在修改前运行测试，了解已有 baseline
cd backend && make lint && make test
cd frontend && pnpm lint && pnpm typecheck
```

### 1.3 代码变更

```bash
# 后端：修改代码，运行格式化
cd backend
make format    # ruff check --fix + ruff format

# 前端：修改代码，运行格式化
cd frontend
pnpm format:write   # Prettier
```

### 1.4 提交前验证

```bash
# 后端完整验证
cd backend && make lint && make test

# 前端完整验证
cd frontend && pnpm lint && pnpm typecheck
BETTER_AUTH_SECRET=local-dev-secret pnpm build  # 如有 UI/路由/认证变更
```

### 1.5 提交和 PR

```bash
git add .
git commit -m "feat: 描述你的变更"
# 或
git commit -m "fix: 修复的问题描述"
# 或
git commit -m "docs: 文档更新"

git push origin feature/your-feature-name
# 然后在 GitHub 创建 Pull Request
```

## 2. 测试策略

### 2.1 后端测试

**运行测试**：
```bash
cd backend
make test           # 运行所有测试（当前约 277 个测试）
make test -- -k "test_name"  # 运行指定测试
make test -- -v     # 详细输出
```

**测试目录**：`backend/tests/`

**关键测试文件**：

| 文件 | 说明 |
|------|------|
| `test_harness_boundary.py` | 确保 `deerflow.*` 不导入 `app.*`（硬边界）|
| `test_docker_sandbox_mode_detection.py` | 从 `config.yaml` 检测沙箱模式 |
| `test_provisioner_kubeconfig.py` | kubeconfig 文件/目录处理 |

**测试框架**：pytest + asyncio

**编写新测试**：
```python
# backend/tests/test_my_feature.py
import pytest
from deerflow.my_module import my_function

def test_my_feature_basic():
    result = my_function("input")
    assert result == "expected_output"

@pytest.mark.asyncio
async def test_my_async_feature():
    result = await my_async_function("input")
    assert result is not None

@pytest.fixture
def my_config():
    return {"key": "value"}

def test_with_fixture(my_config):
    assert my_config["key"] == "value"
```

### 2.2 前端单元测试

```bash
cd frontend
pnpm test
```

### 2.3 前端 E2E 测试

```bash
cd frontend
make test-e2e   # 需要 Chromium；自动构建并启动 Next.js 生产服务器
```

### 2.4 集成测试（使用 DeerFlowClient）

```python
# 使用内嵌客户端进行端到端集成测试
from deerflow.client import DeerFlowClient

def test_full_agent_flow():
    """完整的 Agent 流程测试"""
    client = DeerFlowClient()
    
    events = list(client.stream(
        "计算圆周率到小数点后 5 位",
        config={"configurable": {"model_name": "gpt-4o"}}
    ))
    
    # 验证有输出
    assert len(events) > 0
    
    # 验证有文本响应
    message_events = [e for e in events if e.type == "message"]
    assert any("3.14159" in str(e.data) for e in message_events)
```

## 3. CI/CD 流程

### 3.1 GitHub Actions 工作流

每次 Pull Request 触发以下 CI：

| 工作流文件 | 触发条件 | 说明 |
|-----------|----------|------|
| `backend-unit-tests.yml` | 所有 PR | 后端 lint + 测试 |
| `frontend-unit-tests.yml` | 所有 PR | 前端单元测试 |
| `e2e-tests.yml` | `frontend/` 文件变更时 | 前端 E2E 测试 |

### 3.2 CI 执行步骤（后端）

```bash
uv sync --group dev
cd backend && make lint
cd backend && make test
```

### 3.3 CI 失败排查

常见失败原因：

1. **格式未检查**：运行 `make format`（后端）或 `pnpm format:write`（前端）
2. **测试失败**：运行 `make test` 本地复现
3. **导入边界违反**：`test_harness_boundary.py` 报错，检查 `deerflow.*` 是否导入了 `app.*`
4. **类型错误**：运行 `pnpm typecheck` 查看详情

## 4. 上线前 Checklist

### 4.1 配置检查

```bash
# 确认 config.yaml 使用生产环境配置
cat config.yaml | grep -E "api_key|secret|password"  # 不应有明文密钥

# 确认所有 API Key 通过环境变量配置
export OPENAI_API_KEY="..."
export TAVILY_API_KEY="..."
# 等等

# 确认 config_version 是最新的
grep config_version config.yaml
grep config_version config.example.yaml
# 如果不一致，运行 make config-upgrade
```

### 4.2 安全检查

```bash
# 关闭 API 文档（生产环境）
export GATEWAY_ENABLE_DOCS=false

# 配置 CORS（如果前端和后端不同域）
export GATEWAY_CORS_ORIGINS="https://your-production-domain.com"

# 确保使用 Docker 沙箱（生产环境不要用本地沙箱）
# 在 config.yaml 中设置：
# sandbox:
#   use: aio
```

### 4.3 初始化认证

```bash
# 首次部署：初始化 admin 账户
curl -X POST https://your-domain.com/api/auth/initialize \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@example.com", "password": "your-strong-password"}'
```

### 4.4 性能配置

```bash

# 至少 8 vCPU / 16 GB RAM
# Docker 沙箱需要额外的容器资源

# 推荐配置 summarization 防止长对话 Token 溢出
```

## 6. 常见问题排查

### Q: `make dev` 启动失败

```bash
# 检查端口是否被占用
lsof -i :2026 -i :8001 -i :3000

# 清理之前的进程
make stop

# 检查日志
cat logs/gateway.log
cat logs/frontend.log
cat logs/nginx.log
```

### Q: config.yaml 变更不生效

- **热重载字段**：Gateway 下次请求时自动检测并重载
- **需要重启的字段**：运行 `make stop && make dev`

### Q: 前端构建失败

```bash
# 确认 BETTER_AUTH_SECRET 已设置
export BETTER_AUTH_SECRET=your-secret
BETTER_AUTH_SECRET=$BETTER_AUTH_SECRET pnpm build
```

### Q: 后端 import 报错

```bash
# 检查 Python 依赖
cd backend && uv sync --group dev

# 检查依赖是否安装了指定提供商
# 例如 Google Gemini 需要：
uv add langchain-google-genai
```

### Q: MCP 工具未加载

```bash
# 检查 extensions_config.json 格式
cat extensions_config.json | python3 -m json.tool

# 检查 MCP 服务器是否可执行
which npx   # stdio 类型需要 npx 可用
```
