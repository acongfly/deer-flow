## 2. 总体架构

```
外部 IM 平台（飞书/Slack/Telegram/钉钉）
        │
        ▼ Webhook / Bot 事件
┌───────────────────────────────────────┐
│         Channel 实现层                 │
│  feishu.py / slack.py / telegram.py   │
│  dingtalk.py / wecom.py 等            │
└───────────────┬───────────────────────┘
                │
                ▼ InboundMessage
┌───────────────────────────────────────┐
│         MessageBus（消息总线）         │
│  async pub/sub hub                     │
│  InboundMessage → 队列 → 调度器       │
│  OutboundMessage → 回调 → 渠道        │
└───────────────┬───────────────────────┘
                │
                ▼
┌───────────────────────────────────────┐
│         ChannelManager（核心调度器）   │
│  - 通过 langgraph-sdk HTTP 客户端     │
│    创建/管理线程                       │
│  - 路由命令                            │
│  - Slack/Telegram: runs.wait()        │
│  - 飞书: runs.stream()                │
└───────────────┬───────────────────────┘
                │
                ▼ 内部认证（注入进程本地令牌 + CSRF）
┌───────────────────────────────────────┐
│         Gateway API                    │
│  /api/threads/{id}/runs/stream         │
│  /api/threads/{id}/runs/wait           │
└───────────────────────────────────────┘
```

## 3. 核心组件

### MessageBus（消息总线）

**文件**：`backend/app/channels/message_bus.py`

```python
# 异步 pub/sub hub
InboundMessage → 队列 → 调度器（路由到 ChannelManager）
OutboundMessage → 回调函数 → 各渠道（发送消息）
```

### Store（渠道存储）

**文件**：`backend/app/channels/store.py`

JSON 文件持久化，映射关系：
- `channel:chat_id` → `thread_id`（根对话）
- `channel:chat_id:topic_id` → `thread_id`（线程对话）

### ChannelManager（核心调度器）

**文件**：`backend/app/channels/manager.py`

- 通过 `client.threads.create()` 创建线程
- 路由命令到相应处理器
- **Slack/Telegram**：使用 `client.runs.wait()` 阻塞等待结果
- **飞书**：使用 `client.runs.stream(["messages-tuple", "values"])` 接收增量更新

## 5. 飞书集成详细步骤

### 步骤 1：创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 获取 App ID 和 App Secret

### 步骤 2：配置 Webhook

在飞书应用的事件订阅中配置请求 URL：
```
https://your-domain.com/api/channels/feishu/webhook
```

订阅以下事件：
- `im.message.receive_v1`（接收消息）

### 步骤 3：配置权限

在飞书应用权限管理中添加：
- `im:message:send_as_bot`（发送消息）
- `im:message`（读取消息）

### 步骤 4：本地开发 Webhook 测试

使用 ngrok 或类似工具将本地端口暴露到公网：
```bash
ngrok http 2026
# 将 ngrok URL 配置为飞书 Webhook 地址
```

## 6. 飞书 AI Card 增量更新原理

飞书渠道使用流式更新：每次 Agent 生成新内容时，追踪运行中的 message_id，通过 Patch API 就地更新同一消息卡片，而不是发送多条新消息：

```
Agent 生成第 1 段 → 创建 AI Card → card_message_id 存内存
Agent 生成第 2 段 → Patch 同一卡片（card_message_id）
Agent 生成第 3 段 → 继续 Patch
...
Agent 完成 → 最终 Patch，标记完成
```

---

## 7. 钉钉 AI Card 流式更新

当配置了 `card_template_id` 时，钉钉渠道也支持类似飞书的就地流式更新。不配置时使用普通消息回复。
