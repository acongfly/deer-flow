# 05 — 核心模块：工具系统

> 原始资料：[`backend/CLAUDE.md`](../../backend/CLAUDE.md)

## 1. 工具组装入口

**函数**：`get_available_tools(groups, include_mcp, model_name, subagent_enabled)`  
**文件**：`backend/packages/harness/deerflow/tools/__init__.py`（或 `tools/builtins/`）

工具组装按以下顺序：

```
1. 配置定义的工具（config.yaml via resolve_variable()）
2. MCP 工具（来自已启用的 MCP 服务器，懒加载 + mtime 失效缓存）
3. 内置工具（present_files、ask_clarification、view_image 等）
4. 子代理工具（如果 subagent_enabled=True，则添加 task 工具）
```

## 2. 内置工具

**目录**：`backend/packages/harness/deerflow/tools/builtins/`

| 工具名 | 说明 | 条件 |
|--------|------|------|
| `present_files` | 使输出文件对用户可见（仅限 `/mnt/user-data/outputs`）| 始终加载 |
| `ask_clarification` | 请求用户澄清（由 ClarificationMiddleware 拦截 → 中断）| 始终加载 |
| `view_image` | 读取图片为 base64（仅在模型支持 vision 时添加）| 需 `supports_vision` |
| `setup_agent` | Bootstrap 专用：持久化新自定义 agent 的 `SOUL.md` 和 `config.yaml` | `is_bootstrap=True` 时 |
| `update_agent` | 自定义 agent 专用：在正常对话中持久化对自身的更新 | `agent_name` 已设置且非 bootstrap |

## 4. 社区工具

**目录**：`backend/packages/harness/deerflow/community/`

### 4.1 Tavily（网页搜索 + 抓取）

```
community/tavily/
```

- `web_search`：网页搜索，默认返回 5 条结果
- `web_fetch`：抓取网页内容，限制 4KB
- 需要 `TAVILY_API_KEY` 环境变量

### 4.2 Jina AI（可读性提取）

```
community/jina_ai/
```

- 通过 Jina Reader API 提取网页可读性内容
- 适合处理内容密集的网页

### 4.3 Firecrawl（网页爬取）

```
community/firecrawl/
```

- 通过 Firecrawl API 进行高级网页爬取
- 需要 `FIRECRAWL_API_KEY`

### 4.4 Image Search（图片搜索）

```
community/image_search/
```

- 通过 DuckDuckGo 进行图片搜索

### 4.5 ACP Agent 工具

```
community/aio_sandbox/  (ACP 相关)
```

- `invoke_acp_agent`：调用 `config.yaml` 中配置的外部 ACP 兼容 Agent
- 每线程独立工作区：`{base_dir}/users/{user_id}/threads/{thread_id}/acp-workspace/`
- Lead Agent 通过虚拟路径 `/mnt/acp-workspace/`（只读）访问该目录
- 注意：标准 `codex` CLI 本身不是 ACP 兼容的，需要使用包装器（如 `npx -y @zed-industries/codex-acp`）

## 5. 配置定义工具（config.yaml）

在 `config.yaml` 中使用 `resolve_variable()` 动态配置工具：

```yaml
tools:
  - name: my_custom_tool
    use: my_package.tools:MyCustomTool
    config:
      api_key: $MY_API_KEY
      max_results: 10
```

## 6. 工具在 config.yaml 中的完整配置示例

```yaml
tools:
  - name: tavily_search
    use: deerflow.community.tavily:TavilySearchTool
    enabled: true
    config:
      api_key: $TAVILY_API_KEY
      max_results: 5
      
  - name: jina_reader
    use: deerflow.community.jina_ai:JinaReaderTool
    enabled: true
    config:
      api_key: $JINA_API_KEY
```

## 7. 开发步骤：添加自定义工具

### 方式一：实现 LangChain Tool

```python
# my_tool.py
from langchain_core.tools import BaseTool
from pydantic import BaseModel, Field

class MyToolInput(BaseModel):
    query: str = Field(description="查询内容")

class MyCustomTool(BaseTool):
    name: str = "my_custom_tool"
    description: str = "这个工具用于..."
    args_schema: type[BaseModel] = MyToolInput
    
    def _run(self, query: str) -> str:
        # 实现工具逻辑
        return f"结果: {query}"
    
    async def _arun(self, query: str) -> str:
        return self._run(query)
```

### 方式二：使用 @tool 装饰器

```python
from langchain_core.tools import tool

@tool
def my_simple_tool(query: str) -> str:
    """这个工具用于执行简单查询。"""
    return f"查询结果: {query}"
```

### 注册到工具集

在 `config.yaml` 中引用，或在 `get_available_tools()` 中直接添加。

## 8. 工具错误处理

`ToolErrorHandlingMiddleware` 会捕获所有工具执行异常，转换为 error `ToolMessage`，使 Agent 可以在不崩溃的情况下继续运行：

```python
# 错误会被转换为：
ToolMessage(
    tool_call_id=...,
    content="Error: <exception message>",
    status="error"
)
```

## 9. 工具分组（groups 参数）

`get_available_tools()` 支持按组加载工具，便于在不同场景下（如子代理）加载不同的工具子集：

```python
# 子代理加载全部工具（除了 task 工具，防止递归）
tools = get_available_tools(
    groups=["sandbox", "builtin", "mcp", "community"],
    include_mcp=True,
    model_name="gpt-4o",
    subagent_enabled=False  # 子代理不能再派发子代理
)
```
