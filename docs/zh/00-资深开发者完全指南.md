## 1. 仓库概要与技术栈

DeerFlow 是一个全栈"超级 Agent 框架"，采用前后端分离架构。

### 技术栈

| 层 | 技术 |
|---|---|
| 后端运行时 | Python 3.12 + uv 包管理器 |
| Agent 编排 | LangGraph（状态机图） |
| API 网关 | FastAPI + Uvicorn |
| 前端 | Next.js 16 + React 19 + TypeScript + pnpm |
| 反向代理 | nginx（统一 :2026 端点） |
| 容器化 | Docker + docker-compose |

### 仓库结构

```
deer-flow/
├── backend/                  # Python 后端
│   ├── app/gateway/          # FastAPI 网关 API
│   ├── packages/harness/     # DeerFlow 核心包
│   │   └── deerflow/
│   │       ├── agents/       # Lead agent、记忆、中间件
│   │       ├── sandbox/      # 沙箱执行系统
│   │       ├── subagents/    # 子代理注册与执行
│   │       ├── mcp/          # MCP 集成
│   │       └── tools/        # 工具定义
│   ├── langgraph.json        # 图入口配置
│   ├── pyproject.toml        # Python 依赖
│   └── ruff.toml             # Lint 配置
├── frontend/                 # Next.js 前端
│   ├── src/app/              # Next.js 路由/页面
│   ├── src/components/       # UI 组件
│   ├── src/core/             # 核心逻辑
│   └── src/env.js            # 环境变量模式/验证
├── docker/                   # Docker 配置
├── skills/public/            # 公共技能包
├── .agent/skills/            # 内部 Agent 技能
├── docs/                     # 文档
├── config.example.yaml       # 配置模板
└── Makefile                  # 主入口命令
```

## 2. 快速上手

### 前置条件

```bash
make check
```

所需工具：Node.js >= 22、pnpm >= 10、Python >= 3.12 + uv、nginx

### 一键安装

```bash
cp config.example.yaml config.yaml
# 编辑 config.yaml 填入模型 API 密钥
make install
make dev
```

访问 `http://localhost:2026`。

### 服务端口

| 服务 | 端口 |
|---|---|
| nginx（统一入口） | 2026 |
| frontend | 3000 |
| gateway | 8001 |
| langgraph | 2024 |

## 3. 架构总览

```
用户请求
    │
    ▼
nginx :2026
    │
    ├─► / → frontend :3000 (Next.js UI)
    │
    └─► /api/* → gateway :8001 (FastAPI)
                     │
                     ▼
               LangGraph :2024
                     │
                     ▼
             Lead Agent (状态机)
                     │
            ┌────────┼────────┐
            ▼        ▼        ▼
         工具系统  子代理池   记忆系统
            │
    ┌───────┴───────┐
    ▼               ▼
  沙箱执行        MCP 工具
```

### 请求生命周期

1. 用户在前端发送消息
2. 前端通过 SSE 连接 gateway `/api/chat/stream`
3. Gateway 创建 LangGraph 运行实例
4. Lead Agent 分析请求，路由到工具/子代理
5. 沙箱执行代码/工具调用
6. 结果通过 SSE 流式返回前端

## 4. 核心模块详解

### 4.1 Agent 系统

Lead Agent 基于 LangGraph 状态机构建，是整个系统的核心。

**入口**：`backend/langgraph.json`

```json
{
  "graphs": {
    "agent": "./packages/harness/deerflow/agents:make_lead_agent"
  }
}
```

**核心文件**：
- `deerflow/agents/lead_agent.py` — 主 Agent 逻辑
- `deerflow/agents/middleware.py` — 中间件链
- `deerflow/agents/memory.py` — 长期记忆管理

详见 [03-Agent系统.md](03-Agent系统.md)。

### 4.2 沙箱系统

沙箱提供隔离的代码执行环境，支持三种模式：

| 模式 | 说明 |
|---|---|
| `local` | 本地进程隔离（默认） |
| `docker` | Docker 容器隔离 |
| `e2b` | 云端沙箱（E2B） |

详见 [04-Sandbox沙箱.md](04-Sandbox沙箱.md)。

### 4.3 工具系统

工具是 Agent 可以调用的能力单元：

- **内置工具**：代码执行、文件读写、网络请求等
- **MCP 工具**：通过 Model Context Protocol 加载的外部工具

详见 [05-工具系统.md](05-工具系统.md)。

### 4.4 子代理系统

子代理是可被 Lead Agent 调度的独立执行单元，用于并行处理复杂任务。

**关键特性**：
- 子代理与 Lead Agent 共享沙箱上下文
- 每个子代理是无状态的
- 每轮最多并发 3 个子代理

详见 [06-Subagent子代理.md](06-Subagent子代理.md)。

### 4.5 MCP 集成

DeerFlow 支持 Model Context Protocol，可动态加载外部工具服务器。

**配置示例**（`config.yaml`）：

```yaml
mcp:
  servers:
    - name: my-tool-server
      command: uvx
      args: [my-mcp-package]
```

详见 [07-MCP集成.md](07-MCP集成.md)。

### 4.6 Skills 技能系统

技能是可复用的 Agent 行为包，通过 `SKILL.md` 定义。

**技能目录**：
- `skills/public/` — 公共技能
- `.agent/skills/` — 内部 Agent 使用的技能

**加载机制**：启动时扫描技能目录，将 name + description 注入系统提示，检测到相关意图时加载完整 SKILL.md。

详见 [08-Skills技能系统.md](08-Skills技能系统.md)。

### 4.7 网关 API

FastAPI 网关核心端点：

| 端点 | 方法 | 说明 |
|---|---|---|
| `/health` | GET | 健康检查 |
| `/api/chat/stream` | POST | 流式对话（SSE） |
| `/api/models` | GET | 列出可用模型 |
| `/api/skills` | GET | 列出可用技能 |
| `/api/threads` | GET/POST | 对话线程管理 |

详见 [11-网关API.md](11-网关API.md)。

## 6. 开发工作流

### 日常开发循环

```bash
# 启动全栈服务
make dev

# 后端 lint 和测试
cd backend && make lint && make test

# 前端检查
cd frontend && pnpm lint && pnpm typecheck
```

### 热重载

- **后端**：`uvicorn --reload` 监听 Python 文件变化
- **LangGraph**：`langgraph dev` 模式自动重载
- **前端**：Next.js 默认 HMR

### Git 工作流

```bash
git checkout -b feat/my-feature
# 修改代码
cd backend && make lint && make test
cd frontend && pnpm lint && pnpm typecheck
git add . && git commit -m "feat: add my feature"
```

## 9. 扩展指南

### 添加新工具

1. 在 `backend/packages/harness/deerflow/tools/` 创建工具文件
2. 在 `deerflow/tools/__init__.py` 注册工具

工具遵循 LangChain Tool 规范（`@tool` 装饰器）。

### 添加新技能

1. 在 `skills/public/my-skill/` 创建目录
2. 创建 `SKILL.md`，必需 frontmatter：`name` 和 `description`
3. 可选添加 `scripts/`、`references/`、`assets/` 子目录

### 添加新 API 端点

在 `backend/app/gateway/routers/` 添加路由文件，在 `app/gateway/app.py` 注册路由。

### 集成 MCP 服务器

在 `config.yaml` 添加服务器配置，重启服务后工具自动加载。

### 添加 IM 渠道

详见 [15-IM渠道集成.md](15-IM渠道集成.md)。

## 11. 常见陷阱与注意事项

**前端构建需要 BETTER_AUTH_SECRET**
```bash
BETTER_AUTH_SECRET=your-secret pnpm build
```

**代理环境变量干扰 pnpm**
```bash
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
pnpm install
```

**`make config` 非幂等** — 配置已存在时会中止，后续更新使用 `make config-upgrade`

**子代理最大并发数** — 每轮最多并发 3 个，需要更多时分轮调度

**`pnpm check` 失败** — 不要使用，改用 `pnpm lint` + `pnpm typecheck`
