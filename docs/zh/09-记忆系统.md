## 3. 记忆注入（当前行为）

**函数**：`format_memory_for_injection(memory_data, max_tokens=2000)`

当前注入格式：

```
## User Context
（来自 user.*.summary）
用户是一名 Python 后端开发者，专注于 AI 应用开发

## History
（来自 history.*.summary）
用户最近在开发一个基于 LangGraph 的 Agent 系统

## Facts
（来自 facts[]，按置信度降序，受 Token 预算限制）
- 用户偏好使用 Python 3.12（置信度: 0.9）
- 用户的工作时区是 Asia/Shanghai（置信度: 0.7）
```

**Token 计数**：
- 优先使用 `tiktoken`（`cl100k_base` 编码）
- 若导入失败，回退到 `len(text) // 4`
- 受 `max_injection_tokens` 预算限制（`config.yaml` 中配置）

## 4. MemoryMiddleware 工作流程

**触发时机**：每次对话的 `after_model` 阶段（过滤为用户消息 + 最终 AI 响应）

```
after_model:
  │
  └── 将（用户消息, AI 最终回复）加入异步队列
      │
      ▼ （后台异步处理）
      memory_extractor.extract()
          │
          ├── 调用 LLM 提取新的事实和摘要
          └── 更新 memory.json
```

异步队列确保记忆更新不阻塞对话响应。

```
final_score = (similarity * 0.6) + (confidence * 0.4)
```

计划集成流程：

### 自定义注入格式

修改 `extractor.py` 中的 `format_memory_for_injection()`：

```python
def format_memory_for_injection(memory_data: dict, max_tokens: int = 2000) -> str:
    sections = []
    
    # 自定义：注入项目信息
    if "projects" in memory_data:
        projects = memory_data["projects"]
        if projects:
            section = "## Current Projects\n"
            for p in projects:
                section += f"- {p['name']}: {p['summary']}\n"
            sections.append(section)
    
    # 原有逻辑...
    return "\n\n".join(sections)
```

## 9. 注意事项

- **每用户隔离**：每个用户的记忆存储在独立的 `users/{user_id}/memory.json` 文件中
- **无跨线程合并**：记忆跨线程共享（同一用户的所有对话共享记忆），但线程目录数据是隔离的
- **异步更新**：记忆更新在后台进行，不阻塞对话响应
- **当前状态**：TF-IDF 检索尚未实现（截至 2026-03-10），当前按置信度降序排序
